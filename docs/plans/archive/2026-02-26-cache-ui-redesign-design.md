# 缓存页 UI 重设设计（冷静仪表盘风）

日期：2026-02-26

## 背景
当前 `/cache` 页面信息密度高但层级偏平，缺少“控制平面”节奏感；用户明确要求：
- 以“缓存类型”为核心主视觉，强调类型解释与指标
- 语义签名区独立成段
- 视觉风格为冷静仪表盘风（不引入 Web 字体）

## 目标
- 建立清晰的三段式信息节奏：概览 → 类型 → 语义签名 → 细节面板
- 强化缓存类型“解释+指标+操作”组合，提升可扫读性
- 保持现有功能与接口不变，仅做前端结构与样式重排

## 约束
- 不引入 Web 字体，仅使用系统字体
- 不新增 API 接口或后端修改
- 保持已有操作逻辑（启用/禁用、清空、详情、语义签名刷新等）
- 0.5B 仅做 controller 的约束不变（无业务逻辑变更）

## 信息架构（页面节奏）
1. 头部概览带
   - 标题、说明、缓存后端状态徽标
   - 预热/导出/刷新操作按钮
   - 统计卡片（命中率、条目数、体积、节省 Token、平均耗时等）
2. 缓存类型网格（主视觉）
   - 六类卡片：内容/请求/路由/上下文/Usage/语义
   - 每卡：类型名 + 英文别名 + 说明 + Key 前缀 + 指标（命中率、条目数）
   - 右侧保留启用状态 + 操作（详情/清空）
3. 语义签名观察（独立段）
   - 宽幅卡片承载表格与刷新按钮
4. 细节面板
   - 策略摘要卡 + 缓存内容 Tab（原逻辑保留）

## 视觉与排版
- 风格：冷静仪表盘风，低饱和冷灰底 + 冷蓝绿强调色
- 字体：
  - 标题：`PingFang SC` / `SF Pro Text` / `Segoe UI`
  - 数值：`SF Mono` / `Menlo` / `Consolas`
- 颜色：统一 CSS 变量，减少硬编码
- 动效：轻量入场（淡入 + 轻微上移），卡片可选轻微 stagger

## 数据映射
- 以 `web/src/utils/cache-type-meta.ts` 为类型真相源
- UI 使用 `listCacheTypeMeta()` 生成卡片基础数据
- 统计接口 `/admin/cache/stats` 结果按 `id` 合并 `hitRate`、`entries`、`size`

## 空状态与错误
- 统计接口失败：卡片展示默认 0 值，保留稳定布局
- 语义签名为空：展示 “暂无语义签名数据”
- Memory 降级：保留现有警示条并加强对比度

## 测试与验收
- 单测：`npm run test:unit`
- 类型检查：`npm run typecheck`
- 构建：`npm run build`

## 风险与回滚
- 风险：布局调整可能影响移动端阅读节奏
- 缓解：卡片网格在小屏下改为单列，并保证关键操作可触达
- 回滚：仅前端布局与样式改动，可回退到原 `index.vue`

## 影响范围
- `web/src/views/cache/index.vue`
- `web/src/utils/cache-type-meta.ts`
- `web/src/utils/cache-type-meta.test.ts`
