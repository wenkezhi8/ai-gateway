# AI Gateway 首页 v2 设计文档

日期：2026-02-28

## 1. 目标

在保留开发者快速上手能力的前提下，重做 `http://localhost:8566/` 首页的信息架构与视觉表达，使首页同时具备：

1. 品牌叙事：明确 AI Gateway 的工程定位，而非单纯“功能列表”。
2. 方法可信：将 Superpowers 6 步流程与 TDD 4 阶段作为核心展示。
3. 行动闭环：从“理解价值”到“立即执行”路径更短。

## 2. 参考站抽象

参考来源：`clawd.org.cn`、`openclaw.ai`、`vpsairobot.com/home`、当前本地首页。

抽象出的可复用结构：

1. 首屏价值主张 + 主 CTA（安装/文档/GitHub）。
2. 可信能力区（流程、机制、可验证步骤）。
3. 快速启动区（命令可复制，成本低）。
4. 页尾稳定入口（文档、管理页面、仓库）。

## 3. 方案选择

采用“混合型方案（C）”：

1. 引入 `openclaw` 的品牌叙事强首屏。
2. 保留 `clawd` 的文档/安装友好路径。
3. 保留网关产品的工程化能力矩阵。
4. 将 Superpowers 与 TDD 作为主轴，不做附属说明。

## 4. 页面信息架构

### 4.1 Hero（首屏）

1. 主标题：强调“企业级 AI 开发修复中枢”。
2. 副标题：强调 Superpowers + TDD 的交付方法。
3. 主按钮：`快速开始`、`查看工作流`、`GitHub`。
4. 右侧可视化：请求从 Client 到 Provider Mesh 的流转节点。

### 4.2 Superpowers 工作流区

1. 六步时间线：问题排查、方案讨论、代码修复、回归验证、合规审计、复盘归档。
2. 每步展示“动作 + 输出物”，减少空泛文案。

### 4.3 TDD 执行区

1. 四阶段卡片：`RED`、`GREEN`、`REFACTOR`、`VERIFY`。
2. 每阶段附命令，保证“可执行而非概念化”。

### 4.4 能力矩阵区

1. 质量保障、稳定运行、成本控制、生态兼容四列。
2. 每列 3 个高密度能力点，避免冗长。

### 4.5 快速启动区

1. 保留 Docker / 源码 / API 三标签。
2. 保留命令复制，强化“30 秒启动”提示。

## 5. 交互与表现

1. 配色方向：青蓝 + 橙色信号色，避免“模板感”。
2. 动效方向：首屏卡片轻量上浮，避免重动画。
3. 响应式：`1024px`、`768px` 双断点，手机端单列。

## 6. 测试策略（TDD）

先写失败测试，再改实现：

1. 内容契约测试：
   - 工作流步骤顺序与数量
   - TDD 阶段顺序
   - Hero 主 CTA 完整性
   - 能力矩阵四列存在
2. 实现通过后进行完整验证：
   - `npm run test:unit`
   - `npm run typecheck`
   - `npm run build`

## 7. 范围与风险

1. 改动范围仅限：
   - `web/src/views/home/index.vue`
   - `web/src/views/home/content.ts`
   - `web/src/views/home/content.test.ts`
2. 不触碰后端 API 与路由定义。
3. 风险：视觉改动较大，可能与其他页面主题存在风格差异；可在后续抽象为主题 token 统一。
